Linux學習-基礎篇

電腦個元件的運作

CPU:電腦的大腦

計算單位

  • 除了1 Byte=8 Bits，其他二進位(容量):1024、十進位(速度):1000
  • Bite/Kilo/Mega/Giga/Tera/Peta/Exa/Zetta

頻寬

  • 匯流排寬度x時脈(外頻x倍頻)

超執行緒(HT Hyper-Threading)

  • 假想同時進行兩個程序

主記憶體: 放置正在運算的資料

多通道設計

  • 安插兩支(或四支)主記憶體，提高頻寬

DRAM(Dynamic)與SRAM(Static)

  • 由於DRAM無法達到CPU的時脈，所以需要SRAM幫忙

唯讀記憶體(ROM)

  • 主機板上一些是否要啟動與該功能的各項參數，將被記錄在CMOS晶片上，利用ROM紀錄與讀取CMOS執行所需的程式(BIOS)，不須通電並且寫死在主機板上，但是ROM不可改寫，所以現在的BIOS通常寫入EEPROM(利用特定電壓抹去晶片上的資訊以便改寫)

硬碟: 存放資料

記憶體置換空間(swap): 記憶體不夠用

傳統硬碟

  • SATA: 會受到物理限制速度600Mbyte/s
  • SAS: 比SATA快，但是主機板大多沒有他的連接介面(需外接卡，所以目前還是以SATA為主要介面
  • USB: 也會受到限制:100Mbytes/s

固態硬碟(SSD-Solid State Disk)

  • 不用驅動馬達較不會受到物理限制，快速且省電

主機板: 神經系統

基本設備

  • 處理器插槽: CPU插座
  • 記憶體插槽: Dynamic Random Access Memory
  • ATX電源插槽: 傳遞電
  • BIOS晶片: Basic Input Output System
  • 鋰電池: 關機時供應電
  • 石英振盪器: 提供IC工作時所需的頻率
  • 北橋晶片: 負責連接較快的裝置介面:通常在CPU裡面
  • 南橋晶片: 負責連接較慢得裝置介面

I/O位址: 像是個裝置的門牌

IRQ: I/O到CPU的路徑

連接周邊設備

  • PS/2: 滑鼠與鍵盤的介面，逐漸被USB介面取代
  • USB介面: USB3.0的插槽為藍色
  • RJ-45網路頭: 有內建網路晶片
  • HDML: 有內建顯示晶片

顯示卡: 其資料由CPU控制

由於3D會需要用到龐大的計算，所以現代會內嵌GPU

電源供應器: 一堆電線的鐵盒子

轉換率: 輸出功率/輸入功率(數值越高能源效益越高)

作業系統

作業系統核心(Operating Sysyem):管理電腦所有的活動以及驅動硬體

系統呼叫: 將程式語言轉成核心看得懂的任務函數

核心的功能

系統呼叫介面: 程式開發者與核心的溝通

程序管理: 分配CPU的工作

記憶體管理: 通常核心會提供虛擬記憶體

檔案系統管理: 例如資料的I/O等等工作

Linux的歷史

相容分時系統(CTSS): 讓Terminal(justO/I)連線進入具有運算能力的主機

組合語言(Assembler): 核心程式

Emacs: 一個除錯編輯器

GUL(Graphical User Interface): 圖形使用者介面

由於需求日益加重，所以發展出了XFree86

Linux distribution: Kernel + Softwares + Tools + 可完整安裝程序

讓使用者以光碟/DVD或者透過網路直接安裝/管理Linux系統

主機規劃與磁碟分割

模擬器產生的磁碟檔名為 /dev/vd[a-p] 系列的檔名

為了加速，虛擬機的磁碟由模擬機產生

分割方式

MBR

  • 主要分割與延伸分割最多可以有四筆(硬碟的限制)
  • 延伸分割最多只能有一個(作業系統的限制)
  • 邏輯分割(512bytes)是由延伸分割持續切割出來的分割槽
  • 能夠被格式化後，作為資料存取的分割槽為主要分割與邏輯分割。延伸分割無法格式化
  • 邏輯分割的數量依作業系統而不同，在Linux系統中SATA硬碟已經可以突破63個以上的分割限制

GRT

  • LBA(logical Block Address)0:儲存開機管理程式
  • LBA1:紀錄備分用的GRT分割的放置位置
  • LBA2:這開始每個LBA可以有四個分割紀錄

UEFI效果佳但效能差

磁碟分割的選擇

掛載: 利用一個目錄當成進入點，將磁碟分割槽的資料放置在該目錄下(完整檔名)

主機服務規劃與硬體關係

Linux NAT服務: IP分享、耗電、可額外加裝分析軟體

SAMBA軟體: 加入Windows芳鄰傳送資料

MAIL: 郵件伺服器

Web: WWW伺服器

DHCP: 用戶端自動取的IP

FTP: 一種檔案傳輸協定

目錄的配置

FHS標準

/bin: 放置單人維護模式下還能夠被操作的指令

/boot: 放置開機會使用到的檔案

/dev: 任何裝置與周邊設備以檔案型態存放

/etc: 放置系統主要的設定檔

  • /etc/opt: 第三方協力軟體 /opt 的相關設定檔

/lib: 放置開機時以及在/bin或/sbin底下的指令會呼叫的函式庫

  • /lib/modules/: 放置可抽換式的核心相關模組(驅動程式)

/media: 放置可移除的裝置(e.g.軟碟、光碟、DVD)

/mnt: 暫時掛載某些額外的裝置

/opt: 第三方協力軟體放置的目錄

/run: 放置系統開機後所產生的各項資訊

/sbin: 放置開機、修復、還原系統所需要的指令

/srv: 網路服務啟動之後這些服務所需要取用的資料目錄

/tmp: 一般使用者或者是正在執行的程序暫時放置檔案的地方

/lib<qual>: 放置與 /lib 不同的格式的二進位函式庫

/root: 系統管理員(root)的家目錄

/usr: 詳細內容於下

底下為linux

/lost+found: 系統發生錯誤時放置遺失的片段(ext2/ext3/ext4檔)

/proc: 為『虛擬檔案系統(virtual filesystem)』資料都在記憶體當中，不占硬碟空間

/sys: 同是虛擬的檔案系統，記錄核心與系統硬體資訊較相關的資訊

/usr

/usr/bin/: 放置一般用戶能夠使用的指令

/usr/lib/: 功能=/lib，/lib 就是連結到此目錄中的

/usr/local/: 系統管理員在本機自行下載的軟體(非distribution預設)

/usr/sbin/: 非系統正常運作所需要的系統指令

/usr/share/: 主要放置唯讀架構的資料檔案

/usr/games/

/usr/include/: c/c++等程式語言的檔頭(header)與包含檔(include)

/usr/libexec/: 不被一般使用者慣用的執行檔或腳本(script)

usr/lib<qual>: 功能=/lib<qual>，/lib<qual> 就是連結到此目錄

/usr/src/: 一般原始碼建議放到這，核心原始碼則建議放到/usr/src/linux/目錄下

/var

/var/cache/: 應用程式本身運作過程中會產生的一些暫存檔

/var/lib/: 程式本身執行的過程需要使用到的資料檔案

/var/lock/: 檔案資源一次只能被一個應用程式所使用

/var/log/: 登錄檔放置的目錄

/var/mail/: 個人電子郵件信箱的目錄

/var/run/: 放置某些程式或者是服務啟動後他們的PID

/var/spool/: 放置一些佇列資料(排隊等待其他程式使用的資料

時間更新

modification time(mtime): 檔案的『內容資料』變更

status time (ctime)：狀態 (status)改變時

access time (atime)：該檔案的內容被取用

檔案系統

區塊分類

superblock: 記錄此 filesystem 的整體資訊

inode: 記錄檔案的屬性

block: 實際記錄檔案的內容

Filesystem Description: 描述每個 block group 的開始與結束的 block 號碼

block bitmap : 記錄使用與未使用的 block 號碼

inode bitmap : 記錄使用與未使用的 inode 號碼

檔案系統種類

索引式檔案系統(indexed allocation)

FAT格式系統

Ext2 檔案系統

inode結構

### 其指令

Command

檔案隱藏屬性

chattr ±= [filename]

  • ±=(新增or刪去檔案屬性)
  • A: 避免過度存取
  • S: 同步到硬碟中
  • a: 只能增加不可刪除會修改資料
  • c: 存取時自動壓縮，讀取時自動解壓縮
  • i: 使不能被刪除、改名、設定連結也無法寫入或新增資料
  • s: 被刪時完全移除硬體空間
  • u: 跟s相反

lsattr

  • -a: 顯示隱藏檔的屬性
  • -d: 列出目錄本身屬性非檔名
  • -R: 連同子目錄的資料一併列出來

改變檔案的屬性與權限[-R]

Changegroup(該群組要在etc/group裡)

  • chgrp groupname dirname/filename

Changeowner(該檔案要在etc/passwd裡)

  • chown name dirname/filename

Change Authority(該群組要在etc/group裡)

  • chmod xyz(e.g.770) dirname/filename
  • r:4 w:2 x:1

檔案的特殊權限(權限s標誌)

SUID(4): 像一般user可更改自己passwd，暫時獲得/etc/shadow的權限

  • 僅對binary program有效
  • 需有x權限
  • 僅在執行過程有效
  • 執行者為程式擁有者

SGID(2): 類似SUID但是針對檔案或目錄

  • 僅對binary program有效
  • 需有x權限

SBIT(1): 只能夠針對自己建立的檔案或目錄進行刪除/更名/移動等動作

  • 僅對目錄有效

搜尋

which(針對執行檔和man page)

  • which -a [command]:所有該指令的完整檔名
  • 僅對目錄有效

whereis

  • whereis [-bmsu] [file/directory]

locate [部分檔名]

find 路徑 [option]time](針對時間來搜索檔案)

  • -perm: 有特殊權限的檔案

檔案系統

dumpe2fs -h: 列出 superblock 的資料

blkid: 列出目前系統有被格式化的裝置

ls -li: 看inode的號碼
